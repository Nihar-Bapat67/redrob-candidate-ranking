How it works

FAISS Retrieval

Dense-vector search over the full 100k pool using MiniLM embeddings.

Top-500 Recall

Cheap recall stage narrows 100k candidates down to the 500 most relevant.

Feature Engineering

14 JD-grounded features — domain fit, seniority, product vs services, availability.

LightGBM Ranker

A LambdaRank model distilled from LLM relevance labels re-ranks for precision.

Reasoning Layer

Fact-grounded, rank-aware notes assembled from real profile values — no hallucination.

Honeypot Detection

Deterministic checks flag impossible timelines and inflated skills, then demote them.

The full system retrieves the top-500 from a 100k pool with FAISS, then re-ranks with this LightGBM model. For the live demo (to fit a serverless function) uploaded candidates are scored directly — the same honeypot-removal → feature-matrix → LightGBM → reasoning pipeline, minus the 100k embedding step.
